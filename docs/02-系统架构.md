# 系统架构

## 1. 整体架构

Chet.Admin 采用前后端分离架构，前后端通过 RESTful API 通信：

```
┌──────────────────────────────────────────────────────────┐
│                      浏览器（用户）                         │
└────────────────────────┬─────────────────────────────────┘
                         │ HTTPS
┌────────────────────────▼─────────────────────────────────┐
│            前端（Vue 3 + Ant Design Vue）                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐  │
│  │  路由权限  │  │  状态管理  │  │  API 请求  │  │  视图层  │  │
│  │  Router   │  │  Pinia   │  │  Axios   │  │  Views  │  │
│  └──────────┘  └──────────┘  └────┬─────┘  └─────────┘  │
└──────────────────────────────────┼───────────────────────┘
                                   │ RESTful API (JSON)
┌──────────────────────────────────▼───────────────────────┐
│            后端（.NET 10 WebAPI + Clean Architecture）    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐  │
│  │ 中间件管道 │  │ 控制器层   │  │  应用服务  │  │ 领域模型  │  │
│  │Middleware │→│Controller │→│ Service  │→│ Domain  │  │
│  └──────────┘  └──────────┘  └────┬─────┘  └─────────┘  │
│                                   │                      │
│  ┌────────────────────────────────▼────────────────────┐  │
│  │            基础设施层（Infrastructure）             │  │
│  │   EF Core  │  Redis 缓存  │  Serilog 日志  │  JWT  │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────┬───────────────────────┘
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              ┌──────────┐  ┌──────────┐  ┌──────────┐
              │ SQLite / │  │  Redis   │  │  文件系统  │
              │PostgreSQL│  │ （可选）   │  │  uploads │
              └──────────┘  └──────────┘  └──────────┘
```

## 2. 后端架构（Clean Architecture）

后端解决方案位于 `Chet.Admin.Api/`，遵循 Clean Architecture + 领域驱动设计（DDD），分为四层。**依赖方向只能向内**（外层依赖内层，内层不依赖外层）。

```
Chet.Admin.Api/
├── Chet.Admin.Api/                 # ① 表示层
│   ├── Controllers/                # API 控制器（13 个）
│   ├── Middleware/                 # 中间件
│   ├── Configurations/             # 服务注册扩展方法
│   ├── Filters/                    # 过滤器
│   └── Program.cs                  # 应用启动入口
├── Chet.Admin.Application/         # ② 应用层
│   ├── Chet.Admin.Services/        # 业务逻辑实现
│   └── Chet.Admin.Mapping/         # AutoMapper 映射配置
├── Chet.Admin.Core/                # ③ 核心层（不依赖任何其他层）
│   ├── Chet.Admin.Domain/          # 领域实体
│   ├── Chet.Admin.Contracts/       # 接口契约
│   └── Chet.Admin.Shared/          # 共享类、异常、响应模型
├── Chet.Admin.Infrastructure/      # ④ 基础设施层
│   ├── Chet.Admin.Data/            # EF Core 数据访问
│   ├── Chet.Admin.Caching/         # Redis / NoOp 缓存
│   ├── Chet.Admin.Configuration/   # 强类型配置
│   └── Chet.Admin.Logging/         # 日志配置
└── Chet.Admin.Tests/               # ⑤ 测试层
    ├── Chet.Admin.UnitTests/       # 单元测试
    └── Chet.Admin.IntegrationTests/# 集成测试
```

### 2.1 各层职责

#### ① 表示层（API Layer）

- **Controllers**：处理 HTTP 请求，不含业务逻辑
  - `AuthController` `UsersController` `RolesController` `MenusController`
  - `DepartmentsController` `PermissionsController` `DictionariesController`
  - `DashboardController` `AuditLogsController` `NotificationsController`
  - `FilesController` `OnlineUsersController` `HealthController`
- **Middleware**：横切关注点
  - `AuditLogMiddleware`：记录写操作审计日志
  - `OnlineUserTrackingMiddleware`：刷新用户活跃时间
  - `LogContextMiddleware`：注入日志上下文
  - `RateLimitingMiddleware`：请求限流
- **Configurations**：以扩展方法组织 DI 注册（`ConfigureXxx`）

#### ② 应用层（Application Layer）

- **Services**：业务逻辑实现（`UserService` `AuthService` `RoleService` 等）
- **Mapping**：AutoMapper 配置文件（DTO ↔ Entity 转换）

#### ③ 核心层（Core Layer）

最内层，**不依赖任何其他项目**，定义领域核心：

- **Domain**：领域实体（`UserEntity` `RoleEntity` `MenuEntity` 等）+ `BaseEntity` 基类
- **Contracts**：接口定义
  - 通用：`IRepository<T>`、`IUnitOfWork`
  - 各模块：`IUserService`、`IAuthService`、`IRoleRepository` 等
- **Shared**：`ApiResponse`、`CacheKeys`、自定义异常（`NotFoundException`、`BadRequestException`）

#### ④ 基础设施层（Infrastructure Layer）

- **Data**：`AppDbContext`、`EfCoreRepository<T>`、各模块 Repository 实现、实体 EF 配置
- **Caching**：`RedisCacheService` + `NoOpCacheService`（降级实现）
- **Configuration**：`AppSettings` 强类型配置模型
- **Logging**：`LogContextHelper`、`SensitiveDataLogFilter`

### 2.2 依赖关系

```
表示层 ──→ 应用层 ──→ 核心层 ←── 基础设施层
  │           │            │
  └───────────┴────────────┘ （测试层依赖所有层）
```

- API 层引用 Application 层和 Infrastructure 层
- Application 层只引用 Core 层（通过接口）
- Infrastructure 层引用 Core 层（实现接口）
- 所有层通过依赖注入解耦

## 3. 前端架构（Monorepo）

前端位于 `Chet.Admin.Web/`，是基于 Vben Admin v5.7 的 pnpm Monorepo，主应用为 `web-antd`。

### 3.1 Monorepo 结构

```
Chet.Admin.Web/
├── apps/                    # 应用
│   ├── web-antd/            # ★ 主应用（Ant Design Vue）
│   ├── web-antdv-next/      # Ant Design Vue Next 变体
│   ├── web-ele/             # Element Plus 变体
│   ├── web-naive/           # Naive UI 变体
│   ├── web-tdesign/         # TDesign 变体
│   └── backend-mock/        # Nitro Mock 服务
├── internal/                # 内部工具包
│   ├── node-utils/          # Node 工具
│   ├── tailwind-config/     # Tailwind 配置
│   ├── tsconfig/            # TS 配置
│   └── vite-config/         # Vite 配置
├── playground/              # 示例工程
├── docs/                    # VitePress 文档站点
└── package.json             # 根工作区配置
```

> 业务开发主要在 `apps/web-antd` 中进行。

### 3.2 主应用结构（web-antd）

```
apps/web-antd/src/
├── api/                     # API 请求层
│   ├── core/                # 核心接口（auth/menu/user）
│   ├── system/              # 业务模块接口（10 个）
│   └── request.ts           # Axios 实例 + 拦截器
├── adapter/                 # 组件适配器（form/vxe-table）
├── composables/             # 组合式函数（useDict）
├── layouts/                 # 布局（auth/basic + notification-bell）
├── locales/                 # 国际化（zh-CN/en-US）
├── router/                  # 路由（guard/access/动态路由）
├── store/                   # Pinia 状态（auth）
├── views/                   # 页面
│   ├── _core/               # 内置页面（登录/注册/个人中心/异常页）
│   ├── dashboard/           # 仪表盘
│   ├── system/             # 系统管理（10 个模块）
│   └── demos/              # 示例
├── app.vue                  # 根组件
├── bootstrap.ts             # 应用引导
└── main.ts                  # 入口
```

### 3.3 请求流程

前端请求统一通过 `requestClient`（封装的 Axios 实例）处理：

```
组件调用 API ──→ 请求拦截器（注入 Bearer Token / Accept-Language）
            ──→ 后端 API
            ──→ 响应拦截器（解析 success/data 字段）
            ──→ Token 过期 → 自动刷新 RefreshToken
            ──→ 401/403/404/500 → 统一错误提示
```

## 4. 后端启动流程

`Program.cs` 按六个阶段组织，顺序清晰：

```
① 初始化日志（Serilog）
② 服务注册（DI）── Controllers、Swagger、Database、Redis、
                   AutoMapper、Repositories、Services、
                   FluentValidation、JWT、CORS、MemoryCache
③ 构建 WebApplication
④ 数据库初始化（自动迁移 + 种子数据）
⑤ 中间件管道 ── 异常处理 → 日志上下文 → CORS → 限流
                → SwaggerUI → 认证 → 审计日志 → 在线用户追踪
                → 静态文件 → 控制器映射
⑥ 启动监听
```

## 5. 关键设计原则

- **依赖倒置**：外层依赖内层的接口定义，内层不依赖具体实现
- **单一职责**：每个类、每个方法只做一件事
- **接口抽象**：所有业务逻辑通过接口暴露，便于测试和替换
- **关注点分离**：表示层只处理 HTTP，业务逻辑在应用层，数据访问在基础设施层
- **配置即代码**：DI 注册通过扩展方法组织，集中且可读
