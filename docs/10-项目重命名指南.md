# 项目重命名指南

## 概述

本文档介绍如何使用 `chet-rename` 脚本，将 `Chet.Admin` 快速重命名为你自己的项目名（如 `MyApp.Admin`），一键完成所有文件内容、文件名、文件夹名的替换，并自动校验编译。

适用场景：从模板仓库复制代码后，初始化为自己的项目。

---

## 前置条件

- **Node.js** >= 18
- **.NET SDK** 10
- **pnpm** >= 11（前端启动需要）
- **git**（版本控制与回滚）

确认脚本位置：项目根目录下的 `tools/chet-rename/bin.mjs`。

---

## 快速开始

### 1. 进入项目目录

```bash
cd your-project-root
```

### 2. 预览将要执行的替换（推荐先跑）

```bash
node tools/chet-rename/bin.mjs --dry-run --name MyApp.Admin --github mycompany/MyApp.Admin
```

dry-run 模式不会写入任何文件，只打印：

- 命名映射预览表
- 将要修改的文件数量与匹配数
- 将要重命名的文件 / 文件夹清单

确认无误后再执行下一步。

### 3. 执行重命名

**方式 A：交互式（推荐）**

```bash
node tools/chet-rename/bin.mjs
```

按提示输入项目名、GitHub 仓库地址，确认后执行。

**方式 B：非交互式**

```bash
node tools/chet-rename/bin.mjs --name MyApp.Admin --github mycompany/MyApp.Admin
```

脚本会依次完成：

1. 替换所有文件内容（namespace、配置、文档等）
2. 重命名文件（`.csproj`、`.slnx`、`.db`）
3. 重命名文件夹（从最深层向根目录逐层改）
4. 自动运行 `dotnet build` 校验

---

## 命名风格说明

只需输入一个 PascalCase 名字，脚本自动派生所有命名风格：

| 输入 | 派生项 | 示例 |
| ---- | ------ | ---- |
| `MyApp.Admin` | 项目名（namespace / csproj / 路径） | `MyApp.Admin` |
| | 展示名（标题、版权信息） | `MyApp Admin` |
| | 前端 namespace | `myapp-admin` |
| | Redis 前缀 | `MyAppAdmin:` |
| | 数据库文件 | `MyApp.Admin.db` |
| `mycompany/MyApp.Admin` | GitHub 仓库地址 | `mycompany/MyApp.Admin` |

命名规则：

- 含点号的名字（如 `MyApp.Admin`）：点号在 namespace 中保留，在展示名中变空格，在 kebab-case 中变连字符
- 不含点号的名字（如 `MyApp`）：所有风格自动适配

## 验证

### 后端编译

```bash
dotnet build MyApp.Admin.Api\MyApp.Admin.slnx
```

应看到 `已成功生成` + `0 个错误`。

### 后端启动

```bash
cd MyApp.Admin.Api\MyApp.Admin.Api
dotnet run
```

访问 `http://localhost:5000/swagger`，Swagger 标题应显示为新项目名。

### 前端启动

另开终端：

```bash
cd MyApp.Admin.Web
pnpm install
pnpm dev:antd
```

访问 `http://localhost:5666/`，应看到：

- 浏览器标签标题为新项目展示名
- 登录页 Logo 旁文字为新项目展示名
- 登录后 Dashboard 欢迎语包含新项目展示名

默认登录账号：`admin@example.com` / `Admin@123`。

### 提交到 git

```bash
git add -A
git commit -m "Rename to MyApp.Admin"
```

## 命令参数

| 参数 | 说明 | 示例 |
| ---- | ---- | ---- |
| `--name <name>` | 新项目名（PascalCase，可含点） | `--name MyApp.Admin` |
| `--github <repo>` | GitHub 仓库路径 | `--github user/MyApp.Admin` |
| `--dry-run` | 只预览不写入 | `--dry-run` |

不传 `--name` 时进入交互模式；`--github` 不传时根据 `--name` 自动推导默认值。

## 常见问题

### Q: 执行后 `dotnet build` 报错找不到 csproj？

A: 通常是文件夹重命名顺序异常。运行 `git checkout . && git clean -fd` 回滚后重试。如仍失败，请提交 issue 并附 dry-run 输出。

### Q: 前端启动后白屏或样式异常？

A: 检查 `apps/web-antd/.env` 中的 `VITE_APP_NAMESPACE` 是否已替换为新值（如 `myapp-admin`）。清空浏览器 localStorage 后重试。

### Q: Swagger 标题还是旧的？

A: 后端编译缓存。删除 `bin/` 和 `obj/` 目录后重新 `dotnet build`。

### Q: 登录失败返回 401？

A: JWT Issuer/Audience 已替换为新项目名，前端只携带 token 无需改动。如仍 401，确认后端已重启并清空浏览器 localStorage。

### Q: 想换一个名字重新来过？

A: 在 git 仓库中执行回滚（见下节），然后重新运行脚本。

---

## 回滚

如果重命名过程出错或想重新开始，一键回滚到执行前状态：

```bash
git checkout .
git clean -fd
```

这会丢弃所有改动并删除未跟踪的文件，恢复到上次 commit 的状态。

---

## 注意事项

- **不要在原始 `Chet.Admin` 仓库上直接执行**，脚本会把项目自身改名。正确用法是在复制出的新仓库中运行。
- **脚本会跳过** `tools/`、`node_modules/`、`bin/`、`obj/`、`.git/` 等目录，不会破坏工具自身与构建产物。
- **建议始终先跑 `--dry-run`** 确认映射表无误，再实际执行。
