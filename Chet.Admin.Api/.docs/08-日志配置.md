# 日志配置

## 1. 项目中的 Serilog 配置

### 1.1 配置结构

项目的 Serilog 配置位于 `appsettings.json` 文件中的 `Serilog` 节点：

```json
{
  "Serilog": {
    "MinimumLevel": {
      "Default": "Information",
      "Override": {
        "Microsoft": "Warning",
        "Microsoft.AspNetCore": "Warning",
        "Microsoft.EntityFrameworkCore": "Warning",
        "System": "Warning"
      }
    },
    "WriteTo": [
      {
        "Name": "File",
        "Args": {
          "path": "logs/log-.txt",
          "rollingInterval": "Day",
          "rollOnFileSizeLimit": true,
          "fileSizeLimitBytes": 52428800,
          "retainedFileCountLimit": 7,
          "formatter": "Serilog.Formatting.Compact.CompactJsonFormatter, Serilog.Formatting.Compact",
          "shared": true,
          "flushToDiskInterval": 2
        }
      }
    ],
    "Enrich": [
      "FromLogContext"
    ]
  }
}
```

### 1.2 配置加载方式

Serilog 配置通过以下方式加载：

**Program.cs 中的配置**：
```csharp
builder.Host.UseSerilog((context, configuration) =>
{
    // 清除默认的日志提供程序，避免重复输出
    builder.Logging.ClearProviders();
    
    // 从配置文件读取基础配置
    configuration.ReadFrom.Configuration(context.Configuration);
    
    // 添加丰富的上下文信息
    configuration.Enrich.FromLogContext()
                .Enrich.WithMachineName()
                .Enrich.WithEnvironmentUserName()
                .Enrich.WithProperty("Application", "Chet.Admin")
                .Enrich.WithProperty("Environment", context.HostingEnvironment.EnvironmentName);
    
    // 开发环境特殊配置
    if (context.HostingEnvironment.IsDevelopment())
    {
        configuration.MinimumLevel.Debug()
                   .WriteTo.Console(
                       outputTemplate: 
                           "[{Timestamp:HH:mm:ss}] [{Level:u4}] {Message:lj}{NewLine}" +
                           "    └─ Properties: {Properties}{NewLine}" +
                           "{Exception}",
                       theme: Serilog.Sinks.SystemConsole.Themes.AnsiConsoleTheme.Code);
    }
    
    // 生产环境特殊配置
    if (context.HostingEnvironment.IsProduction())
    {
        configuration.MinimumLevel.Information()
                   .WriteTo.Console(new Serilog.Formatting.Compact.CompactJsonFormatter());
    }
});
```

## 2. 配置项详细说明

### 2.1 基础配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `MinimumLevel.Default` | 默认日志级别 | `Information` |
| `MinimumLevel.Override` | 针对特定命名空间的日志级别覆盖 | `{"Microsoft": "Warning"}` |

### 2.2 输出配置（WriteTo）

#### 2.2.1 文件输出

```json
{
  "Name": "File",
  "Args": {
    "path": "logs/log-.txt",
    "rollingInterval": "Day",
    "rollOnFileSizeLimit": true,
    "fileSizeLimitBytes": 52428800,
    "retainedFileCountLimit": 7,
    "formatter": "Serilog.Formatting.Compact.CompactJsonFormatter, Serilog.Formatting.Compact",
    "shared": true,
    "flushToDiskInterval": 2
  }
}
```

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `path` | 日志文件路径 | `logs/log-.txt` |
| `rollingInterval` | 日志滚动间隔 | `Day`（按天滚动） |
| `rollOnFileSizeLimit` | 是否按文件大小限制滚动 | `true` |
| `fileSizeLimitBytes` | 单个日志文件大小限制 | `52428800`（50MB） |
| `retainedFileCountLimit` | 保留日志文件数量 | `7`（保留7天） |
| `formatter` | 日志格式化器 | `Serilog.Formatting.Compact.CompactJsonFormatter` |
| `shared` | 允许多进程共享日志文件 | `true` |
| `flushToDiskInterval` | 刷新到磁盘的间隔（秒） | `2` |

### 2.3 日志增强（Enrich）

| 增强器 | 说明 |
|--------|------|
| `FromLogContext` | 从日志上下文添加属性 |
| `WithMachineName` | 添加机器名称（代码中配置） |
| `WithEnvironmentUserName` | 添加环境用户名（代码中配置） |

## 4. 日志上下文工具

### 4.1 LogContextHelper

项目提供了 `LogContextHelper` 工具类，用于便捷地添加日志上下文信息。

#### 使用示例

**在中间件中添加请求上下文**：
```csharp
public async Task InvokeAsync(HttpContext context)
{
    using (LogContextHelper.WithRequest(
        context.TraceIdentifier,
        context.Request.Method,
        context.Request.Path.Value ?? ""))
    {
        if (context.User?.Identity?.IsAuthenticated == true)
        {
            var userId = context.User.FindFirst("sub")?.Value ?? "unknown";
            var userName = context.User.Identity.Name ?? "unknown";
            
            using (LogContextHelper.WithUser(userId, userName))
            {
                await _next(context);
            }
        }
        else
        {
            await _next(context);
        }
    }
}
```

**在服务中添加业务上下文**：
```csharp
public async Task CreateOrderAsync(int orderId, string userId)
{
    using (LogContextHelper.WithProperty("OrderId", orderId))
    using (LogContextHelper.WithUser(userId, "张三"))
    {
        _logger.LogInformation("开始创建订单");
        // 业务逻辑...
        _logger.LogInformation("订单创建成功");
    }
}
```

#### 可用方法

| 方法 | 说明 | 参数 |
|------|------|------|
| `WithUser(userId, userName)` | 添加用户上下文 | 用户ID、用户名 |
| `WithRequest(requestId, method, path)` | 添加请求上下文 | 请求ID、HTTP方法、路径 |
| `WithProperty(propertyName, value)` | 添加单个属性 | 属性名、属性值 |
| `WithProperties(dictionary)` | 批量添加多个属性 | 属性字典 |

## 5. 敏感信息过滤

### 5.1 SensitiveDataLogFilter

项目提供了 `SensitiveDataLogFilter` 过滤器，用于自动过滤日志中的敏感信息。

#### 默认过滤的敏感关键词

- password（密码）
- token（令牌）
- secret（密钥）
- apikey（API密钥）
- authorization（授权信息）
- creditcard（信用卡号）
- ssn（社会安全号）
- privatekey（私钥）

#### 启用方法

在 `SerilogConfiguration.cs` 中取消注释：

```csharp
// 添加敏感信息过滤器（可选）
// 自动过滤包含密码、令牌等敏感信息的日志
configuration.Filter.With<SensitiveDataLogFilter>();
```

#### 过滤效果

**过滤前**：
```csharp
_logger.LogInformation("用户登录: {Password}", "MySecret123");
// 会输出到日志文件
```

**过滤后**：
```csharp
_logger.LogInformation("用户登录: {Password}", "MySecret123");
// 不会输出到日志文件（被过滤）
```

## 6. 日志中间件

### 6.1 LogContextMiddleware

项目提供了 `LogContextMiddleware` 中间件，自动为每个请求添加上下文信息。

#### 注册中间件

在 `Program.cs` 中：

```csharp
// 添加日志上下文中间件，为所有请求添加上下文信息
app.UseLogContext();
```

#### 功能特性

- ✅ 自动添加请求ID、HTTP方法、路径
- ✅ 如果用户已认证，自动添加用户信息
- ✅ 记录请求处理时间
- ✅ 支持请求链路追踪

## 7. 实际应用示例

### 7.1 在控制器中使用

```csharp
[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
    private readonly ILogger<UsersController> _logger;
    private readonly IUserService _userService;

    public UsersController(ILogger<UsersController> logger, IUserService userService)
    {
        _logger = logger;
        _userService = userService;
    }

    [HttpGet("{id}")]
    public async Task<ActionResult<UserDto>> GetUser(int id)
    {
        _logger.LogInformation("正在获取用户ID为 {UserId} 的信息", id);

        try
        {
            var user = await _userService.GetUserByIdAsync(id);
            
            if (user == null)
            {
                _logger.LogWarning("未找到ID为 {UserId} 的用户", id);
                return NotFound(new { message = "用户不存在" });
            }

            _logger.LogInformation("成功获取用户ID为 {UserId} 的信息", id);
            return Ok(user);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "获取用户ID为 {UserId} 的信息时发生错误", id);
            throw;
        }
    }
}
```

### 7.2 在服务中使用

```csharp
public class UserService : IUserService
{
    private readonly ILogger<UserService> _logger;
    private readonly IUserRepository _userRepository;
    private readonly IMapper _mapper;

    public async Task<UserDto> CreateUserAsync(UserCreateDto createUserDto)
    {
        _logger.LogInformation("开始创建新用户，用户名: {UserName}", createUserDto.Name);

        var user = _mapper.Map<User>(createUserDto);
        user.PasswordHash = BCrypt.Net.BCrypt.HashPassword(createUserDto.Password);

        var createdUser = await _userRepository.AddAsync(user);
        await _userRepository.SaveChangesAsync();

        var userDto = _mapper.Map<UserDto>(createdUser);
        
        _logger.LogInformation("成功创建新用户，用户ID: {UserId}，用户名: {UserName}", 
            createdUser.Id, createUserDto.Name);

        return userDto;
    }
}
```

## 8. 结构化日志记录最佳实践

### 8.1 使用占位符

```csharp
// 好的做法 - 使用占位符
_logger.LogInformation("用户 {UserId} 在 {Timestamp} 执行了 {Action} 操作", userId, timestamp, action);

// 避免的做法 - 字符串插值
_logger.LogInformation($"用户 {userId} 在 {timestamp} 执行了 {action} 操作");
```

### 8.2 记录异常信息

```csharp
try
{
    // 一些可能引发异常的操作
}
catch (Exception ex)
{
    _logger.LogError(ex, "执行操作时发生错误，用户ID: {UserId}，操作类型: {OperationType}", userId, operationType);
    throw;
}
```

### 8.3 使用日志上下文

```csharp
using (LogContextHelper.WithUser(userId, userName))
```

## 10. 性能考虑

### 10.1 条件日志记录

```csharp
// 在性能敏感的代码中使用 IsEnabled 检查
if (_logger.IsEnabled(LogLevel.Debug))
{
    var expensiveValue = ExpensiveMethod();
    _logger.LogDebug("昂贵计算的结果: {ExpensiveValue}", expensiveValue);
}
```

### 10.2 异步日志记录

Serilog 默认是线程安全的，并且大多数 Sink 都支持异步写入，这有助于减少日志记录对应用程序性能的影响。

### 10.3 文件写入优化

- `shared: true` - 允许多进程共享日志文件
- `flushToDiskInterval: 2` - 每2秒刷新到磁盘，平衡性能和可靠性

## 11. 故障排除

### 11.1 日志重复输出

如果日志输出两遍，检查：

1. 是否清除了默认的日志提供程序：
   ```csharp
   builder.Logging.ClearProviders();
   ```

2. 是否在配置文件和代码中重复配置了 Console 输出

### 11.2 验证配置文件

确保配置文件语法正确，并且 Serilog 相关的 NuGet 包已正确安装：

- `Serilog.AspNetCore`
- `Serilog.Enrichers.Environment`
- `Serilog.Enrichers.Thread`

通过合理配置和使用 Serilog，您可以获得强大的日志记录能力，帮助您更好地监控和调试应用程序。
